SemanticAPI class

Used for make the initial configurations, make some operations on and mainly store one resource in the dataset.

The contructor sctructure will be showed bellow:

                                    class SemanticAPI{
                                        constructor(config){
                                            ...
                                            this.config = config
                                            this.defaultParams = {
                                                headers: {
                                                  'Content-Type': 'application/json'
                                                }
                                            }
                                        }
                                    }

The methods of the class are:

                                    //Used to persist a resouce in dataset
                                    saveResource(resource)

                                    //Used to delete a resource from dataset given the resource URI
                                    deleterResource(resourceUri)

                                    //Delete the workspace where the SemanticAPI object is configured to
                                    deleteGraph()

                                    //Delete an property of a resource given both URI's
                                    deleteProperty(resourceUri, propertyUri)

                                    //Retrieve the resource given his URI
                                    getResource(resourceUri)

                                    /* Retrive all resources the corresponde to the match of equality of property and
                                     * value. This function consider if all value are exactly like the passed if
                                     * isExactly (a boolean) is true. Otherwise is considered all values that contais
                                     * the passed value
                                     */
                                    getResources(propertyUri, value, isExactly)

                                    /* Update a property of a resource given the URI of the resource, of the property
                                     * and the new value respectivaly
                                     */
                                    updateProperty(resourceUri, propertyUri, newValue)

                                    /* Add a property to one resource given the the resourcec(in this case firstly the
                                     * resource must be retrieved of server), the vocabulary prefix and an property
                                     * created previously
                                     */
                                    addProperty(resource, vocabPrefix, property)

                                    /* Since one resource was retrieved by the server and had various changes this
                                     * function make all changes persistent in the dataset
                                     */
                                    commitChanges()

                                    //Verify if one resource already exists on the dataset
                                    alreadyExistsInDatabase(resource)

                                    //Get the local name of the resource retrieved
                                    getResourceLocalName(resource)

                                    //Get all resources of same given type in all given workpaces
                                    getResourcesByType(workspaces, type)

Resource class

This class has funtions que enables the assembly of a resource

The contructor sctructure will be showed bellow:

                                    class Resource {
                                        constructor(name, prefix, uri){
                                        this.vocabularies   = {}
                                        this.type           = name
                                        this.prefix         = prefix
                                        this.about          = uri
                                        }
                                        ...
                                    }

The methods of the class are:

                                    //Assign the type of the resource
                                    setType(type)

                                    //Assign the prefix of the resource
                                    setPrefix(prefix)

                                    //Assign the prefix of the resource
                                    setAbout(about)

                                    //Assign the vocabularies of the resource
                                    setVocabularies(vocabularies)

                                    //Assign a new vocabulary to the resource
                                    //This function creates a new instance of a vocabulary e add to the resource
                                    addVocabulary(vocabPrefix, uri)

                                    //Verify if a vocabularie prefix already exists in the list of passed vocabularies
                                    vocabExists(vocabPrefix, vocabularies)

                                    //Add a new property to one vocabulary
                                    addProperty(vocabPrefix, property)

                                    //Add a new property with a language to one resource
                                    addLanguage(language)

                                    //Add latitude, longitude, date and time of creating of the resource
                                    addCoordinatesDateTime

Property class

This class only has a constructor of a property to add to a resource

The contructor sctructure will be showed bellow:

                                    class Property {
                                        constructor(propertyName, value, asResource, subPropertyOf){
                                        this.propertyName   = propertyName
                                        this.value          = value
                                        this.asResource     = asResource
                                        this.subPropertyOf  = subPropertyOf
                                        }
                                    }

Mounting a resource 
                                    const resource = new Resource('contactData', 'contact', 'http://contactmail.com#Person')
                                    resource.addVocabulary('schema', 'http://schema.org/')
                                    resource.addVocabulary('vcard', 'http://www.w3.org/2006/vcard/ns#')
                                    property = new Property('hasValue', 'email@email.com', false, 'email')
                                    resource.addProperty('vcard', property)
                                    property1 = new Property('client', 'YahooMail', false, 'email')
                                    resource.addProperty('vcard', property1)
                                    property2  = new Property('name', 'Lucimar da Silva Souza', false, '')
                                    resource.addProperty('schema', property2)
                                    property3 = new Property('about', 'Assunto do email', false, '')
                                    resource.addProperty('schema', property3)
                                    property4 = new Property ('telephone', '73991809696', false, '' )
                                    resource.addProperty('schema', property4)
                                    property5 = new Property('text', 'Mensagem do e-mail', false, '')
                                    resource.addProperty('schema', property5)
    In this example is shown the assembly of a new resource. First is used the function addVocabulary, from Resource, that have three the parameters: type, prefix and uri respectively Then instances of Vocabulary object are created.
    To create one Vocabulary is necessary to pass his prefix and namespace uri. Later are created new instances of Property. Each property has: name, value, a boolean that determines if the property will be stored as a Resouce (the uri of the object), and the last parameter determines the parent property. If this parameter is filled means that this property is actualy a property of a blank node belonging to the resource. Then is called the function addProperty of Resource class passing as parameters the vocab prefix and the property created early.
Saving a resource 
                                    api.saveResource(resource)

                                    .then(json => alert(`Recurso salvo com sucesso\nSeu workspace é: ${json.workspace}        
                                               \nO ID do recurso é: ${json.resourceId}`);)
                                    .catch(error => alert(`Problem trying save the resource: \n${error.message}`))
The saveResource function saves one resource later that it is mounted. This function belongs to the SemanticAPI class. Is used the object of this class created previously to call the function passing as parameter the resource created before. This function returns a promise to treat the cases of success or fail in the operation. In the promise treatment events can be captured one object that have two variables: workspace and resourceId.

Once captured one specific resource we can edit it adding vocabularies, properties, add a language or the coordinates and time. The resource searched can be captured in the promise treatment event. And then can be edited as in example above. But to make all changes permanent in dataset should be called the function commitChanges, belonging to the SemanticAPI class also. Should be given one special attention to the function addProperty wich in this case belong to SemanticAPI class but exits another way to add a new property calling the addProperty of Resource. In the first case the difference is that the first parameter should be the resource that will have the property added. Finnaly commitChanges is called to make the changes permanent in dataset.
